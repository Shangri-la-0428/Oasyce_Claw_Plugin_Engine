## Design Context

### Users
- **Who**: Data creators, AI developers, node operators, network participants
- **Context**: Managing data rights, AI capabilities, and consensus participation on a decentralized protocol
- **Job to be done**: Register/monetize data assets, publish/invoke AI services, run validator/arbitrator nodes, track earnings and network state
- **Emotional goals**: Security & trust (data is safe, everything transparent), Control & professionalism (operating a precision instrument), Cyberpunk & cutting-edge (inhabiting a future network)

### Brand Personality
- **3 words**: Precise · Restrained · Cutting-edge (精密 · 克制 · 前沿)
- **Voice**: Technical but clear. No marketing fluff. Every word earns its place.
- **Tone**: Confident, calm authority. Like a well-engineered instrument panel.

### Aesthetic Direction
- **Style**: Constructivism × Bauhaus Functionalism × Retro-Futurism
- **Palette**: Pure black/white/gray. Semantic color ONLY for status (green=success, red=error, yellow=warning, blue=info). Never decorative color.
- **Theme**: Dark mode default, light mode support. Both use the same grayscale philosophy.
- **Typography**: 4-level strict hierarchy (Display 36px / Body 15px / Caption 12px / Label 11px). Monospace for technical data (IDs, hashes, prices).
- **References (feel right)**: Linear, Vercel (极简黑白、信息密度高、工程师审美), Stripe/Notion (清晰友好、文档感强), Terminal/CLI (纯文本美学、黑客气质、monospace 主导)
- **Anti-references**: Colorful crypto dashboards, gradient-heavy Web3 UIs, gamified interfaces, anything with gratuitous animation or decoration

### Design Principles
1. **Function is beauty** — Every element serves a purpose. If it doesn't inform or enable action, remove it.
2. **Density over decoration** — Pack useful information into clean layouts. Whitespace is structural, not decorative.
3. **Restrained palette** — Black, white, gray. Color is a signal, never a style choice. When color appears, it means something.
4. **Progressive disclosure** — Show essentials first, reveal complexity on demand. Collapsible sections, expandable forms, layered detail.
5. **Monospace is identity** — Technical data (keys, IDs, amounts, hashes) always in monospace. It signals precision and trust.

### Tech Stack
- Preact 10 + @preact/signals (3KB, fine-grained reactivity)
- Pure CSS custom properties (zero CSS dependencies)
- Vite 8 (instant builds)
- TypeScript strict mode
- Bilingual i18n (zh/en) via computed signals

### Component Conventions
- Cards: 1px border, 8px radius, 32px padding. Hover changes border color only.
- Buttons: 40px height primary, 32px compact. Ghost variant for secondary actions.
- Inputs: 44px height, transparent background, 1px border. Focus = border darkens.
- KV pairs: flex space-between, 14px vertical padding, separated by 1px top border.
- Animations: 0.12-0.3s duration, cubic-bezier(0.4, 0, 0.2, 1). Subtle translateY/opacity only. No bounce, no spring.
- Touch targets: min 44×44px for all interactive elements.

### Responsive Breakpoints
- Desktop: max-width 640px centered
- Tablet (≤768px): full width, 24px card padding
- Mobile (≤600px): 16px gap, 24px display font, stacked KV layout
- Small phone (≤375px): tighter toast margins
