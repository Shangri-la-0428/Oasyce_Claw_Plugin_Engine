## Design Context

### Users
- **Primary:** Data providers/creators who want to monetize their data assets — focused on earnings, security, and rights protection
- **Secondary:** AI developers/researchers who need training data and AI capabilities — focused on efficiency and precision
- **Context:** Professional users managing valuable digital assets in a decentralized marketplace. They expect the same reliability as financial infrastructure.

### Brand Personality
**Precise. Restrained. Trustworthy.**

The interface should feel like a well-engineered instrument panel — every element earns its place, nothing is decorative. Data is the asset class, and the UI must reflect that gravity.

### Emotional Goals
- **Precision & Trust** — like a financial terminal: accurate, reliable, serious
- Users should feel their data and funds are safe
- Confidence in every transaction, clarity in every status

### Aesthetic Direction
- **Visual tone:** Minimal, technical, monochrome with semantic color only for status
- **References:** Linear, Vercel (information density), Stripe (clarity), Terminal (monospace precision)
- **Anti-references:** Colorful crypto dashboards, gradient-heavy Web3 UIs, gamified interfaces, gratuitous animation
- **Theme:** Dark mode default, light mode supported. Both use grayscale philosophy.

### Design Principles

1. **Earn every pixel** — No decoration. Every element must serve a purpose. If it doesn't inform or enable action, remove it.

2. **Monospace signals trust** — All technical data (IDs, hashes, prices, addresses) uses monospace. This communicates precision and machine-verified accuracy.

3. **Semantic color only** — Green/red/yellow/blue are reserved exclusively for status (success/error/warning/info). All other UI is grayscale. Never use color for decoration.

4. **Subtle motion, never spectacle** — Animations are 0.12-0.3s max, cubic-bezier easing, translateY + opacity only. Respect prefers-reduced-motion. No bounce, spring, or spin.

5. **Data density over whitespace** — Professional users want information, not breathing room. Prioritize showing useful data over aesthetic spacing, but maintain clear visual hierarchy.

--bg-0: #0a0a0a    --fg-0: #e8e8e8    --border: #1a1a1a

--green: #5a9e7a   --red: #b85c5c   --yellow: #b8a44a   --blue: #6a8fb5

### Typography
```
Sans: -apple-system, BlinkMacSystemFont, 'Inter', 'Segoe UI', sans-serif
Mono: 'JetBrains Mono', 'SF Mono', 'Fira Code', monospace (13px fixed)

Display: 36px / 300 weight / -0.025em tracking
Body:    15px / 400 weight / 1.7 line-height
Caption: 12px / 400 weight / 0.04em tracking
Label:   11px / 600 weight / 0.1em tracking
```

### Spacing & Layout
```
--gap: 28px   --nav-h: 48px   --max-w: 640px
--r-s: 2px    --r-m: 6px      --r-l: 8px
Card padding: 32px (desktop), 24px (tablet), 16px (mobile)
Touch target: 44x44px minimum
```

### Components
- **Cards:** 1px border, 8px radius, hover changes border only
- **Buttons:** Primary (fg on bg), Ghost (transparent + border), Danger (red text)
- **Inputs:** 44px height, transparent bg, 1px border, focus darkens border
- **Animations:** 0.12-0.3s, cubic-bezier(0.4, 0, 0.2, 1), reduced-motion respected
