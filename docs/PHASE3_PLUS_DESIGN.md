## 1. Scanner + Inbox（资产自动发现与确认）

### 用户旅程

```
用户日常工作 → Scanner 后台扫描产出物
    ↓
发现可注册资产（Skill、文件、数据集……）
    ↓
AI 自动生成：描述 · 标签 · 敏感度评级 · 建议定价
    ↓
推入 Inbox 待确认队列
    ↓
用户在 Dashboard 看到：✅ 批准 / ❌ 拒绝 / ✏️ 编辑
    ↓
批准 → 自动调用 oasyce register
```

### 信任梯度

| 等级 | 行为 | 适用 |
|------|------|------|
| L0 | 每笔确认 | 新用户默认 |
| L1 | 低价值自动，高价值确认 | 用了一段时间 |
| L2 | 全自动，异常才通知 | 完全信任 |

### Inbox 两个 Tab

- **待注册**：AI 发现的可注册资产，确认后上链
- **待购买**：网络上的购买请求 / AI 推荐的能力，确认后成交

### 架构位置

Scanner 和 Inbox 是**客户端功能**，放在 PE 层。协议层零改动。

## 2. 协议自消费能力（Protocol as Consumer）

### 核心洞察

协议运行需要算力（验证、指纹、标签生成……），这些工作本身就是 capability。**协议是自己能力市场的第一个客户。**

### 协议内务能力清单

| 能力 | 谁调用 | 谁付钱 | 谁干活 |
|------|--------|--------|--------|
| `asset_similarity_check` | 协议（注册时） | 注册者（从注册费扣） | 验证节点 |
| `asset_tag_generate` | 协议（注册时） | 注册者 | AI 推理节点 |
| `fingerprint_embed` | 协议（交付时） | 买方（从交易费扣） | 计算节点 |
| `fingerprint_scan` | 协议（巡查时） | 协议基金（从罚款回补） | 巡查节点 |
| `dispute_arbitrate` | 协议（争议时） | 败诉方 | 仲裁节点 |
| `storage_proof` | 协议（定期） | 数据主人（存储费） | 存储节点 |

### 实现方式

不需要新的资产类型或协议机制。节点注册 capability → 协议作为调用方 → 正常的调用-托管-结算流程。市场竞争自动优化基础设施质量。

## 3. 巡查执法（Bounty Hunter）

### 机制

```
巡查节点注册 fingerprint_scan 能力
    ↓
持续爬取公开平台（GitHub、HuggingFace、公开数据集……）
    ↓
匹配已注册资产的语义指纹
    ↓
发现疑似泄漏 → 提交证据 → 自动发起争议
    ↓
仲裁确认 → 泄漏者押金没收
    ↓
赏金分配：巡查节点 40% · 数据主人 40% · 协议 20%
```

### 威慑体系

| 层级 | 机制 | 状态 |
|------|------|------|
| 经济威慑 | 质押 + 泄漏扣押金 | ✅ 已有 |
| 指纹追踪 | 语义水印溯源到买家 | ✅ 已有 |
| 网络巡查 | 赏金猎人爬取公开平台 | 📋 本次规划 |
| 信誉惩罚 | 泄漏者信誉归零 + 永久记录 | ✅ 已有 |

### 诚实的边界

- ✅ 高代价威慑 + 事后追责 + 自动检测
- ❌ 无法物理阻止数据被复制（没人能）
- ❌ 无法检测完全重写/重新表述的数据（语义指纹极限）

## 4. 分布式存储（OAS-Storage）

### 现状

Phase 1-3：本地存储，够用。

### 长期规划

```
原始数据 → 加密 → 纠删码分片（10/14）→ 分散到 14 个节点
                                          ↓
                          任意 10 片可恢复 · 单片无意义 · 4 节点掉线不丢数据
```

### 经济模型

| 角色 | 行为 | 经济 |
|------|------|------|
| 数据主人 | 付存储费 | 支出 OAS |
| 存储节点 | 存碎片 + 定期存储证明 | 赚存储奖励 |
| 掉线/作恶节点 | 证明失败 | 扣质押金 |

### 存储证明（Proof of Storage）

协议定期随机挑战：要求节点返回指定碎片的指定字节段。正确 → 继续赚钱。失败 → 扣押金 + 碎片重建分配给新节点。

### 实现策略

- **短期**：本地存储（当前）
- **中期**：对接 IPFS（快速可用，不重复造轮子）
- **长期**：自建 OAS-Storage 标准（完全自主）

存储服务本身也是 capability，走正常的调用结算。
